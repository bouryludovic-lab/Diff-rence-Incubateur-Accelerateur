Introduction à l'Automatisation

01

Qu'est-ce que l'automatisation ?

Vous avez probablement déjà entendu le mot « automatisation » partout. Mais concrètement, de quoi parle-t-on ? En une phrase : automatiser, c'est confier à un logiciel une tâche que vous faisiez à la main.

Copier-coller des données d'un outil à un autre, envoyer un email de bienvenue à chaque nouveau client, mettre à jour un tableau de suivi après une vente — tout ça, un logiciel peut le faire à votre place, sans erreur, 24h/24.

✈️ L'analogie du pilote automatique

Imaginez un avion. Le pilote décolle et atterrit manuellement — ce sont les tâches complexes qui demandent du jugement. Mais en croisière, il active le pilote automatique : l'avion maintient son cap, son altitude et sa vitesse tout seul. L'automatisation, c'est exactement ça : vous gardez le contrôle sur les décisions importantes, et vous déléguez les tâches répétitives à la machine.

Ce qu'on peut automatiser (et ce qu'on ne peut pas)

Automatisable vs Non automatisable

La règle est simple : si une tâche suit toujours les mêmes étapes, se déclenche à partir d'un événement prévisible, et ne nécessite pas de jugement humain — elle est automatisable.

💡
Et l'IA dans tout ça ?Avec l'IA, la frontière recule. Des tâches qui demandaient du jugement (classifier un email, résumer un document) deviennent automatisables. On en reparlera au Module 3.8.

03

Les niveaux d'automatisation

Toutes les automatisations ne se ressemblent pas. On peut les classer en 3 niveaux, du plus simple au plus avancé.

Niveau 1 — Simple
Un déclencheur, une action. La base de toute automatisation.
Ex : « Nouveau contact → Email de bienvenue »
Niveau 2 — Multi-étapes
Plusieurs actions enchaînées, avec conditions et branchements.
Ex : « Formulaire → CRM + Slack + Email personnalisé »
Niveau 3 — Agent IA
L'IA prend des décisions dans le workflow selon le contexte.
Ex : « L'IA lit l'email, classifie, et répond automatiquement »
💡
Vous allez passer les 3 niveauxCette formation vous guide progressivement : scénarios simples → workflows multi-étapes → agents IA au Module 3.8.

❌ L'automatisation ne remplace pas les humains — elle leur enlève les tâches répétitives pour qu'ils se concentrent sur ce qui compte.
2. Quel est le principal avantage de l'automatisation pour une petite équipe ?
✅ Bravo ! Temps libéré + zéro erreur + scalabilité = les 3 piliers du gain.

Module 3.3

Les API : le langage des machines

Comprendre comment les outils communiquent entre eux grâce aux API — le fondement de toute automatisation.

📚 4 leçons⏱️ ~30 min🎯 Quiz de validation
07

Qu'est-ce qu'une API ?

API signifie Application Programming Interface. En clair : c'est la porte d'entrée qui permet à deux logiciels de se parler.

🍕 L'analogie du restaurant

Vous (l'application) ne pouvez pas aller en cuisine directement. Vous passez votre commande au serveur (l'API). Le serveur transmet à la cuisine (le serveur distant). La cuisine prépare et le serveur vous rapporte la réponse.

Quand Make envoie un email via Gmail, il envoie une requête à l'API de Gmail en disant : « Envoie cet email à cette adresse. » Et l'API exécute.

💡
Vous n'avez PAS besoin de coder des APIMake s'occupe de tout. Mais comprendre le concept vous aidera à débugger et à connecter des outils plus avancés.
08

Les méthodes HTTP

Il existe 4 « verbes » principaux dans le langage HTTP — les 4 actions de base qu'on peut faire avec des données.

MéthodeActionAnalogieExemple
GETLire / RécupérerConsulter le menuRécupérer la liste de vos contacts
POSTCréer / EnvoyerPasser une commandeCréer un nouveau contact dans le CRM
PUTModifierChanger votre commandeMettre à jour l'email d'un contact
DELETESupprimerAnnuler la commandeSupprimer un contact obsolète
✅
Les 4 verbes dans votre boîte à outils !GET = lire, POST = créer, PUT = modifier, DELETE = supprimer. Dans la prochaine leçon, on aborde l'authentification.
09

Authentification API

Quand Make appelle l'API de Gmail ou Slack, ces services doivent vérifier que vous avez le droit d'accéder aux données. C'est l'authentification.

🔑 L'analogie du badge d'accès

Certains badges ouvrent toutes les portes (admin), d'autres seulement certains étages (permissions limitées). L'authentification API fonctionne pareil : vous présentez vos « identifiants » au service.

🔑 API Key
Un code secret unique que vous copiez-collez. Simple mais basique.
Ex : « sk-abc123xyz789 »
🔐 OAuth 2.0
Vous autorisez Make via une fenêtre de connexion.
Ex : « Connecter avec Google »
🎫 Bearer Token
Un jeton temporaire généré après connexion.
Ex : « Bearer eyJhbGci... »
🔐 IMPORTANT : Ne partagez jamais vos clés API

Votre clé API est comme le code de votre carte bancaire. Si quelqu'un l'obtient, il pourra utiliser votre compte.

Tester une API dans Make

L'exemple : récupérer la météo d'une ville

On va utiliser Open-Meteo — une API 100% gratuite, sans inscription, sans clé API.

🎬 Démo : appeler l'API Open-Meteo depuis Make

On configure le module HTTP pour interroger l'API météo en live et récupérer la température actuelle de Paris.

  1. Créez un nouveau scénario dans Make
  2. Ajoutez un module → recherchez « HTTP » → sélectionnez « Make a request »
  3. Dans le champ URL, collez :
https://api.open-meteo.com/v1/forecast?latitude=48.8566&longitude=2.3522¤t_weather=true
  1. Dans Method, sélectionnez GET
  2. Laissez les Headers et Body vides
  3. Cliquez sur « Run once »

Make affiche la réponse de l'API. En cliquant sur la bulle verte, vous verrez :

{"current_weather": {"temperature": 18.5, "windspeed": 12.3, "weathercode": 2, "time": "2026-03-10T14:00"}}
✅
Bravo, votre premier appel API !Vous venez de faire un GET sur une API et de récupérer la température en temps réel. Passons aux webhooks.

Qu'est-ce qu'un webhook ?

Avec une API classique, c'est vous (ou Make) qui allez chercher l'information. Avec un webhook, c'est le service qui vous prévient quand quelque chose se passe.

🔔 L'analogie de la sonnette

Sans webhook, vous ouvrez la porte toutes les 5 minutes pour vérifier si le livreur est passé. Avec un webhook, vous avez une sonnette : le livreur sonne quand il arrive.

⚡ Instant Trigger (intégré)🔗 Custom Webhook (universel)
📦 Où le trouverModule natif de l'app (ex : Typeform, Stripe)Module Webhooks → Custom webhook
🔧 ConfigurationAutomatique — connecter votre compteManuelle — copier l'URL dans le service
🔗 URL webhookCachée, gérée par MakeVisible, à copier-coller
🎯 Quand l'utiliserQuand le module de l'app a le ⚡Quand l'app n'a pas de module natif
💡
Dans les deux cas, c'est du temps réelLa seule différence, c'est qui configure le lien : Make automatiquement (⚡) ou vous manuellement (🔗). Privilégiez toujours l'Instant trigger ⚡ quand il existe.
12

PollingWebhook

✅
Vous maîtrisez la différence !Polling = Make va vérifier. Webhook = le service prévient Make. Le webhook est toujours préférable quand c'est possible.

Qu'est-ce que le MCP ?

Le MCP (Model Context Protocol) est un standard ouvert créé par Anthropic qui permet aux modèles d'IA (comme Claude) de se connecter directement à vos outils et bases de données.

🌍 L'analogie de l'interprète universel

Le MCP, c'est un interprète universel pour l'IA : il lui permet de « parler » avec n'importe quel outil — Google Sheets, Slack, HubSpot — dans un langage standardisé.

Les modules Make : anatomie d'un scénario

Maîtriser les briques fondamentales : triggers, actions, routers, variables, filtres et itérateurs.

📚 5 leçons⏱️ ~35 min🎯 Quiz de validation
19

Les triggers (déclencheurs)

Chaque scénario Make commence par un trigger (déclencheur). C'est l'événement qui lance votre automatisation.

🚦 L'analogie du feu vert

Le trigger, c'est le feu vert de votre scénario. Tant qu'il est rouge, rien ne bouge. Dès qu'il passe au vert, tout le scénario se met en mouvement.

🔄 Watch (Polling)
Make vérifie périodiquement si quelque chose de nouveau est apparu.
Ex : « Watch Emails » — vérifie Gmail toutes les 15 min
⚡ Instant (Webhook)
Le service prévient Make en temps réel.
Ex : « New Submission » Typeform — instantané
⏰ Schedule
Le scénario se lance à des heures précises.
Ex : « Tous les lundis à 9h »
20

Après le trigger, viennent les actions — les modules qui font quelque chose.

TypeCe que ça faitExemples

UpdateMettre à jourModifier le statut d'un deal
DeleteSupprimerSupprimer une ligne, archiver une tâche

21

Les routers (aiguillages)

Un router permet de séparer le flux en plusieurs chemins. Chaque chemin peut avoir ses propres conditions et actions.

🚦 L'analogie du carrefour

Selon le type de véhicule, le feu dirige vers des routes différentes. Le router fait pareil : il dirige les données vers le bon chemin selon des conditions.

💡
Un router peut avoir autant de branches que nécessaireSi aucune condition n'est définie, les données passent dans toutes les branches.
22

Variables et mapping

Quand vos données passent d'un module à l'autre, elles sont transportées par des variables. Le mapping, c'est l'action de dire à un module : « Utilise cette donnée qui vient du module précédent. »

🧩 L'analogie du champ [Prénom]

« Bonjour [Prénom], merci pour votre commande. » Les champs entre crochets sont des variables remplacées par la vraie valeur. Dans Make, c'est pareil.

📝 Valeur fixe
Texte en dur, identique à chaque exécution.
Ex : « Bienvenue chez Studeria »
🧩 Variable mappée
Cliquez sur une variable d'un module précédent pour l'insérer dynamiquement.
Ex : l'email Typeform → champ « To » Gmail
🔀 Mix des deux
Texte fixe + variables pour personnaliser.
Ex : « Bonjour [Prénom], commande [Numéro] confirmée »

Dépanner une connexion cassée

🔑 Token expiré
L'autorisation a expiré (souvent après 90 jours).
Solution : cliquez sur « Reauthorize »
🔒 Mot de passe changé
Make ne peut plus se connecter avec l'ancien mdp.
Solution : recréez la connexion
🚫 Permissions révoquées
Vous avez retiré l'accès de Make dans les paramètres.
Solution : re-autorisez Make
⏰ Compte désactivé
Votre abonnement au service a expiré.
Solution : réactivez votre compte
  1. Identifiez la connexion en erreur dans Connections
  2. Essayez « Reauthorize » — ça résout 80% des cas
  3. Si nécessaire, supprimez et recréez la connexion
  4. Sélectionnez la nouvelle connexion dans les modules concernés
